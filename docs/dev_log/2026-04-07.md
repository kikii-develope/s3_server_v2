## 작업 개요
- v7.1 미디어 업로드/변환 시스템의 테스트 안정화 및 운영 경로 안전장치 보강
- Swagger 테스트 환경 분리, 다중 변환 업로드 API 추가, CYA 정책 확정, WebDAV 루트 경로 자동 분기 반영

## 코드 변경 상세

### 1) Swagger/로컬 테스트 환경 정리
- `docker-compose.swagger-test.yml` 신규 추가
  - 로컬 Swagger 테스트용 서버 + Redis 분리 실행
  - 테스트 포트 환경변수(`SWAGGER_TEST_PORT`) 적용
- `src/config/swagger.js`
  - 서버 목록에 로컬 URL 우선 노출
  - `SWAGGER_SERVER_URL` 지원
- `index.ts`
  - `/health` 엔드포인트 추가

### 2) 업로드 변환 API 확장
- `POST /webdav/upload-convert-multiple` 신규 구현
  - 파일 배열(`files`)을 순차 처리
  - 파일별 처리 결과 + summary(`queued/converted/skipped/reused/failed`) 반환
- 공통 처리 함수(`processConvertUploadFile`)로 단일/다중 경로 통합

### 3) CYA 파일 처리 정책 확정
- `file`/`xxd`/`ffprobe` 분석 결과 CYA가 일반 ffmpeg 입력 컨테이너가 아님을 확인
- 정책 확정:
  - CYA는 변환 제외
  - 원본 확장자 그대로 NAS 업로드
  - 상태는 `skipped`로 기록

### 4) WebDAV 루트 경로 안전장치
- 신규 유틸: `src/utils/webdavRootPath.js`
  - `WEBDAV_ROOT_PATH` 명시 시 우선 사용
  - 미명시 시 `NODE_ENV` 자동 분기
    - production -> `www`
    - 그 외 -> `kikii_test`
- `webdavController`, `webdavClient`, `videoQueue`에서 공통 유틸 사용하도록 정리
- `/webdav/directory/{path}` 조회 시 `/www` 하드코딩 사용 문제 제거

### 5) 에러 응답 품질 개선
- 다중 업로드 결과에서 메시지가 빈 문자열로 내려오지 않도록 기본 메시지 보강

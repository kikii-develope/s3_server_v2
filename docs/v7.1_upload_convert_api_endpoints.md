# v7.1 Upload Convert API 엔드포인트 상세

## 1) POST /webdav/upload-convert
단일 파일 업로드 + 변환 엔드포인트입니다.

### 요청
- Content-Type: `multipart/form-data`
- 필수 필드
  - `file`: 파일 1개
  - `path`: 업로드 상대 경로 (예: `wh/case-001`)
- 선택 필드
  - `domain_type`
  - `domain_id`

### 처리 규칙
- 이미지(`jpg/jpeg/png/...`)는 Sharp로 변환 후 NAS 업로드
- 영상(`mp4/avi/mov/...`)은 BullMQ Job 큐에 적재 후 비동기 변환
- `cya`는 벤더 전용 포맷 특성상 변환하지 않고 원본 업로드(`skipped`) 처리
- 그 외 문서/바이너리는 변환 없이 원본 업로드
- 동일 해시 중복 파일이면 기존 변환 결과 재사용

### 주요 응답
- `200`: 변환 완료 또는 변환 불필요 업로드 완료 또는 중복 재사용
- `202`: 영상 변환 작업 접수 완료 (`jobId`, `statusUrl` 포함)
- `409`: 동일 파일 처리 중 충돌
- `400`: 요청값 오류
- `500`: 서버 내부 오류

---

## 2) POST /webdav/upload-convert-multiple
다중 파일 업로드 + 변환 엔드포인트입니다.

### 요청
- Content-Type: `multipart/form-data`
- 필수 필드
  - `files`: 파일 배열 (최대 20개)
  - `path`: 업로드 상대 경로
- 선택 필드
  - `domain_type`
  - `domain_id`

### 처리 방식
- 각 파일마다 `upload-convert`와 동일 규칙을 적용
- 응답은 파일별 결과 배열 + 요약 통계(`queued`, `converted`, `skipped`, `reused`, `failed`)를 반환

### 응답 예시 필드
- `summary.total`: 전체 파일 수
- `summary.queued`: 영상 큐 등록 건수
- `summary.converted`: 즉시 변환 완료 건수(주로 이미지)
- `summary.skipped`: 변환 없이 원본 업로드 건수
- `summary.reused`: 기존 변환본 재사용 건수
- `summary.failed`: 실패 건수
- `results[]`: 각 파일별 상세 결과 (`originalName`, `status`, `message`, `metadataId`, `jobId`, `path` 등)

---

## 3) GET /webdav/convert-status/{id}
변환 메타데이터 ID 기준으로 변환 상태를 조회합니다.

### 요청
- Path parameter
  - `id`: `file_convert_metadata.id`

### 응답 필드
- `convertStatus`: `pending | processing | uploading | completed | failed | skipped`
- `jobId`: 영상 큐 Job ID (영상인 경우)
- `failureType`: `retryable | permanent`
- `error`: 실패 메시지
- `convertedPath`: NAS 저장 경로
- `retryCount`: 재시도 횟수
- `updatedAt`, `completedAt`

---

## 상태 전이 요약
- 이미지: `pending -> processing -> uploading -> completed`
- 영상: `pending -> (queue) -> processing -> uploading -> completed`
- 문서/기타: `pending -> skipped` (원본 업로드)
- 실패: `failed` + 에러 정보 기록

---

## 운영 체크포인트
- WebDAV 루트 경로는 기본적으로 `NODE_ENV` 기준 자동 분기
  - `production` -> `www`
  - 그 외(`development`, 로컬 테스트) -> `kikii_test`
- `WEBDAV_ROOT_PATH`를 명시하면 자동 분기보다 우선 적용됨
- `CONVERT_DB_*` 환경변수 미설정 시 변환 메타 DB 기록/상태 관리가 실패할 수 있음
- Swagger에서 호출 전 상단 Server URL이 로컬 테스트 포트(`http://localhost:8991` 등)인지 확인

# 🛠️ NAS 미디어 자동 변환 시스템 v7.1 상세 구현 명세 (2026-04-06)

본 문서는 2026년 4월 6일 구현 완료된 NAS 미디어 자동 변환 시스템(v7.1)의 핵심 로직과 모듈별 상세 내용을 설명합니다.

---

## 1. 시스템 개요
본 시스템은 기존 WebDAV 업로드 방식에 **'자동 변환'** 파이프라인을 추가하여, 업로드 시 이미지(WebP 등)와 영상(MP4/H.264)을 자동으로 변환하고 NAS에 저장합니다. 9가지 주요 Edge Case(레이스 컨디션, 서버 크래시, 좀비 파일 등)에 대한 방어 로직이 포함되어 있습니다.

추가 반영 사항:
- 변환 상태 조회 API: `GET /webdav/convert-status/:id`
- 변환 전용 업로드 입력 검증(`path` 필수)
- WebDAV 디렉토리 생성 시 405/409 응답 허용 처리
- `/www/www/...` 경로 중복 방지 정규화
- 다중 변환 업로드 API: `POST /webdav/upload-convert-multiple`
- CYA 파일 정책: 변환 제외 + 원본 업로드(`skipped`)
- WebDAV 루트 자동 분기: `production=www`, 그 외=`kikii_test` (환경변수 명시 시 우선)

## 2. 모듈별 상세 상세 내역

### 📂 유틸리티 레이어 (`src/utils/`)
- **`dbRetry.js`**: 데이터베이스 데드락이나 일시적인 연결 유실 시 최대 3회 재시도하며, 지수 백오프(Exponential Backoff)를 적용해 DB 부하를 조절합니다.
- **`tempCleaner.js`**: AWS 서버(`/tmp/upload-temp`)의 임시 파일을 안전하게 삭제합니다. 서버 시작 시와 주기적(10분 간격)으로 오래된 파일을 자동 정리합니다.
- **`dbLogger.js`**: `console.log` 대신 DB 테이블(`convert_log`)에 로그를 기록합니다. 로그는 7일 후 자동으로 삭제되어 디스크 용량을 보존합니다.
- **`errorClassifier.js`**: 발생한 에러를 '재시도 가능(네트워크 등)'과 '영구 실패(파일 손상 등)'로 분류하여 자원 낭비를 방지합니다.

### ⚙️ 서비스 레이어 (`src/services/`)
- **`mediaProcessor.js`**: `sharp`(이미지)와 `fluent-ffmpeg`(영상)를 사용한 실제 변환 엔진입니다. 영상 변환 시 10분 타임아웃 및 강제 종료(SIGKILL) 로직이 적용되어 무한 루프를 방지합니다.
- **`dedup.js`**: SHA-256 스트림 해시를 계산하여 이미 완료된 동일 파일이 있을 경우 변환을 건너뛰고 기존 파일을 재사용(Deduplication)합니다.
- **`videoQueue.js`**: `BullMQ` 기반의 영상 변환 워커입니다. CPU 코어 수에 따른 동시성 제어와 DB Lock(`locked_at`)을 통해 여러 워커가 동일 파일을 중복 처리하는 것을 방지합니다.
- **`watchdog.js`**: 시스템의 수호자 역할을 합니다. 30분 초과 정체 작업(Stuck) 실패 처리, 업로드 중 서버 종료로 남은 NAS 임시 파일(`.__uploading__`) 정리, 큐 유실(Zombie) 복구 등을 수행합니다.

### 🌐 WebDAV 레이어 (`src/services/web_dav/`)
- **`webdavClient.js` (확장)**: `atomicUpload` 기능을 추가했습니다. NAS에 직접 업로드하지 않고 임시 이름으로 먼저 전전송한 후, 완료 시점에만 이름을 변경(Rename)하여 네트워크 끊김 시 깨진 파일이 저장되는 것을 방지합니다.
- **경로 정규화**: `/www` 중복 경로를 방지하도록 `ensureDirectory`에서 prefix 정규화 처리.

### 📑 컨트롤러 및 라우터 (`src/controller/`, `src/router/`)
- **`webdavController.js`**: `uploadWithConvert` 함수를 통해 전체 흐름을 제어합니다. 해시 계산 -> 중복 체크 -> DB 레코드 생성 -> 이미지(동기) / 영상(비동기 큐) 분기 처리를 담당합니다.
- **`webdavController.js`**: 상태 조회 `getConvertStatus` 추가.
- **`webdavController.js`**: 다중 처리 `uploadWithConvertMultiple` 추가.
- **`webDavRoutes.js`**: 신규 엔드포인트 `POST /webdav/upload-convert`, `POST /webdav/upload-convert-multiple`, `GET /webdav/convert-status/:id` 추가.

## 3. 적용된 안전 장치 (Edge Case Defense)
| 항목 | 설명 |
| :--- | :--- |
| **Atomic Upload** | 임시 파일 업로드 후 Rename 방식을 사용하여 데이터 무결성 보장 |
| **Race Condition 방지** | `content_hash` UNIQUE 제약 조건과 DB 트랜잭션/락 활용 |
| **Zombie Recovery** | Watchdog이 주기적으로 상태 불일치(uploaded 상태 정체 등)를 감지하여 자동 복구 |
| **Disk Guard** | 업로드 시작 전 서버 디스크 여유 공간(파일 크기 x 2 + 500MB) 사전 체크 |
| **경로 정규화** | `/www/www/...` 중복 경로 방지 |

---
**주의**: 테스트 전 `init-convert-db.sql`을 실행하여 로컬 또는 테스트용 DB 서버에 필요한 테이블을 반드시 생성해야 합니다.

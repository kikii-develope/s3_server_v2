# CYA 파일 분석 및 변환 전략

대상 파일: `1818_P9901_박영순_01.CYA`

## 1) 로컬 분석 결과
실행 결과 요약:
- `file`: `Composite Document File V2 Document, Cannot read section info`
- 헤더 시그니처: `D0 CF 11 E0 A1 B1 1A E1` (OLE2 Compound File)
- `ffprobe`: `Invalid data found when processing input`

해석:
- 일반 MP4/MOV/AVI 컨테이너가 아니라, 블랙박스 벤더 전용 OLE 기반 컨테이너일 가능성이 높음
- ffmpeg가 직접 디멀티플렉싱하지 못해 즉시 변환이 불가한 상태

## 2) 용량 절감/변환 가능한 방법
우선순위 순으로 권장:

1. 벤더 전용 뷰어/추출 툴 사용
- CYA에서 표준 영상(mp4/avi)으로 먼저 export
- 이후 서버에서 ffmpeg 재압축 적용

2. 벤더 SDK/CLI 연동
- 블랙박스 제조사 SDK가 있으면 서버 파이프라인에 변환 단계 추가
- 가능하면 원본 메타(GPS/시간/채널) 보존 옵션 사용

3. 서버 정책 분기(현재 적용)
- CYA는 영상 큐 변환 대상에서 제외
- CYA는 업로드 즉시 원본 확장자 그대로 NAS에 저장(`skipped` 상태)

4. 압축 아카이브 보관
- 변환 불가 시 `zip/zstd` 같은 파일 압축으로 저장 용량 절감 시도
- 단, 이미 내부 압축/암호화된 포맷이면 절감률이 낮을 수 있음

## 3) 현재 운영/테스트 정책
- CYA는 원본 보존 업로드(무손실)로 처리
- 일반 영상(mp4/avi/mov 등)만 비동기 변환 대상
- CYA 관련 상태는 `skipped`로 기록되어 후속 모니터링 가능
- 벤더 툴 또는 SDK 확보 전까지 CYA 자동 변환은 비활성 유지

## 4) 결론
현재 샘플 CYA는 ffmpeg 단독으로 직접 변환 불가입니다.
즉시 서비스 안정성을 우선하면 **CYA는 원본 확장자 그대로 업로드**하는 정책이 안전합니다.
